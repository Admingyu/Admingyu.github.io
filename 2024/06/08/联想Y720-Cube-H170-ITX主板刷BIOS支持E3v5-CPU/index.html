联想Y720 Cube / H170 ITX主板刷BIOS支持E3v5 CPU

2024-06-08 814 字

组itx找到了这个主板：


查了下原来是联想的一款游戏主机的主板型号是Y720 Cube，主板是H170芯片组，
故入，上了G3930CPU，用了一年，最近想学虚拟化和k8s，两核的G3930不能玩，想着升级一下cpu，查了下资料：

Intel CPU接口：LGA1151，支持7代CPU,Y720 cube原配的cpu是i7 -7700

但是7700太贵了，要500多，想着都是1151针，看能不能上E3毕竟E3便宜


考虑到功耗，集显需求，综合下来选择了E3-1225V5,76包邮

魔改BIOS，使主板支持E3：

提取BIOS

  1. 官网下载Y720 Cube Biso升级程序

    • 使用7zip解压，提取最新版本的BIOS文件IMAGEO36.bin

  2. 如果找不到官网BIOS，只能用编程器读取主板上bios程序

修改BIOS

  1. 使用CoffieTime 打开bios文件，如图，修改ME到最新版本，Replace，并将状态改为Disabled

  2. 把下面的Fix和Patch全解锁

  3. 查了下微码，目标CPU微码已经在BIOS里面了，所以只需要解锁就行，就没有改这块，不清楚为什么有两个506E3，一个是C2，另一个是A6希望知道的大佬讲解下

  4. vBios和GOP也升级到最新版本

  5. 转到Extra选项卡，这三个全解锁成绿色

  6. 点击Save image将修改后的bios保存

备份旧的BIOS，刷入新的BIOS

编程器硬件方面，我使用CH341A编程器，软件方面我是用NeoProgrammer，也可以使用CH341A programmer

  1. 拆下主板上的bios芯片

    夹在编程器上，插到另一台电脑USB接口

  2. 备份片上bios，如果检测芯片型号不成功，可以试一下手动查bios芯片上的丝印，然后通过Search按钮选择
    注意，这这样的话要确保能读出来bios数据，且数据不能是空白(FF/00)

  3. 写入新的bios

  4. 打开修改后的bios

  5. 清空芯片

  6. 写入bios

  7. 校验写入数据

完成之后将bios芯片焊回主板，安回CPU，Ram 开机测试

开机滴滴声报错问题

这样刷完之后，开机会有两声滴滴报错，在bios logo页面显示无法读取序列号错误，虽然不影响正常开机但是还是想做的完美点，可以进系统后通过程序写入，程序就是官网下载的bios升级程序解压出来的那个
ANMIDEWIN.exe,根据这个指引，写入相关信息 how-to-update-the-machine-type-and-model-mtm-system-serial-number

# To flash machine type and model: 
ANMIDEWIN.exe /sp "String"
# To flash serial number: 
ANMIDEWIN.exe /ss "String"
# To flash system brand ID: 
ANMIDEWIN.exe /sv "String"

String替换为相应的值

这样开机就没有报错了。
